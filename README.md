# Прототип приложения-калькулятора для чисел в различных системах счисления (2-Z Notation Scale: + - * /)
![Скриншот](./screenshots/2ZNotationScale_1.jpgs)

## Назначение программы
Арифметические действия над числами в системах счисления от двоичной до тридцатипятиричный.

## Средства разработки
- **Язык программирования**: C++. 
- **Среда разработки**: Borland C++ Builder 6. 
- **Программа справочной системы**: Microsoft Help Workshop.

## Описание интерфейса программы
![Скриншот](./screenshots/2ZNotationScale_2.jpgs)

1. Поле ввода аргументов и отображения результатов.
По итогам некоторых операций может содержать сообщения об ошибках, вроде «ПЕРЕПОЛНЕНИЕ!» или «ПРЕВЫШЕНИЕ!».

2. Цифры для формирования аргументов ввода из алфавита текущей системы счисления.
Аналогичными для них являются клавиши консоли, соответствующие символьным представлениям отображаемых цифр.
В этом случае необязателен ввод в режиме Caps Lock.
Доступными являются только те цифры, которые входят в алфавит текущей системы счисления.

3. Смена знака текущего аргумента из поля ввода.
В том случае, если аргумент нулевой или поле содержит сообщения об ошибках, то знак не влияет на отображаемую в данном поле информацию.

4. 5. 6. 7. Символы бинарных арифметических операций (сложение, вычитание, умножение, деление) над числами.
При их возникновении происходит операция над числом из памяти и введёнными после знака операции аргументом, после чего результат заносится в память и выводится на экран, в поле ввода-вывода.

8. Операция обнуления.
Значение текущего аргумента из поля ввода, но не из памяти, сбрасывается и устанавливается в нуль.

9. Операция равенства.
В поле выводится значение аргумента из памяти.
А с началом новой операции обновляется число, хранящееся в памяти, куда записывается аргумент из поля ввода-вывода, оказавшийся там до появления символа новой операции.

10. Удаление последнего символа с конца у аргумента из поля вывода.
Если текущее число состоит лишь из одного значащего символа или поле ввода содержало сообщение об ошибке, то после нажатия данной кнопки аргумент устанавливается в нуль.

11. Вызов справочной системы калькулятора.

12. Вызов окна, информирующего о разработке данной программы.

13. Область смены текущей системы счисления, где можно выбрать основание новой системы счисления от двух до тридцати пяти.
При некорректном вводе система считается десятичной.
При маленьком основании, оно становится равным минимальному, то есть двум.
Превысившее допустимое значение основание, устанавливается в максимальное – в тридцать пять.

## Описание библиотеки классов
Не все предполагавшиеся классы были реализованы.
Данное приложение является прототипом.

### 1. T_Cl_Digital_Array
Это некоторое разрядное пространство, не принадлежащее ни к одной системе счисления, где хранятся символы цифр и организованы операции на уровне работы с массивом.

### 2. T_Cl_Notation_Scale
Система счисления – совокупность правил существования по законам в её микромире – алфавит, допустимые значения и операции тестирующего характера.

### 3. T_Cl_Notation_Scale_Number_Order
Число, имеющее массив разрядов, определённых в системе счисления: жизнедеятельные процессы их симбиоза клеточного массива в индивидуальной системе счисления.
Взаимодействие с другими подобными существами описаны в рамках этикета определённых операций, таких, как сравнение, присвоение.
Сложение, вычитание, умножение и деление. Миграция в другие системы счисления.

### 4. T_Cl_Notation_Scale_Number_Whole_Part
Крупное целое число, обладающее видоизменёнными поведенческими особенностями своего предка, переопределённые для его крупных габаритов.
Класс находится в состоянии развития своего перспективного продолжения и сущетвования.

### 5. T_Cl_Notation_Scale_Fractional_Part
Предполагаемый ход мутации целой части в дробную часть где разряды постоянно инвертируются для адекватного существования. 

### 6. T_Cl_Notation_Scale_Number
Вполне сформированное число, с определённой целой и дробной частями и порядком, для которого выращен весь пёстрый букет всевозможных операций, но в силу большой разрядности всех его компонентов, не очень удобное для представления пользователю, не контактирующее с внешним миром.

### 7. T_Cl_ Notation_Scale_Suitable_Number
Улучшенное поколение предыдущего числа, активно взаимодействующее с пользователем, гибкое и лояльное в своём зрительном представлении.
